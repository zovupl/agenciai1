Podstawy — Agenci AI
Agent AIAI Agent

Program, który autonomicznie realizuje cel: planuje kroki, podejmuje decyzje i wykonuje zadania, korzystając z dostępnych narzędzi. Więcej niż chatbot.

ChatbotClassic chatbot

Odpowiada na pojedyncze pytania według skryptu lub modelu. Nie planuje i nie wykonuje wieloetapowych działań samodzielnie.

AutonomiaAutonomy

Zdolność agenta do działania bez ręcznego zatwierdzania każdego kroku — sam dobiera kolejne działania prowadzące do celu.

AdaptacyjnośćAdaptability

Dostosowanie sposobu działania do zmieniających się warunków i nowych informacji w trakcie realizacji zadania.

System multi-agentMulti-agent system

Kilku agentów współpracujących nad wspólnym celem — każdy odpowiada za swój fragment (np. researcher + redaktor + korektor).

Typy agentów
Agent reaktywnyReactive agent

Reaguje na bieżące zdarzenia według prostych reguł, bez długoterminowego planowania. Szybki, do prostych, powtarzalnych reakcji.

Agent deliberatywnyDeliberative agent

Planuje i analizuje wiele kroków przed działaniem. Najlepszy, gdy zadanie wymaga strategii i rozważenia scenariuszy.

Agent hybrydowyHybrid agent

Łączy szybkie reagowanie na zdarzenia z planowaniem i analizą — najczęściej spotykany w praktyce.

Projektowanie agenta
Prompt systemowySystem prompt

Stała instrukcja definiująca agenta: rola, cel, kontekst, zasady, ograniczenia i format odpowiedzi. Fundament zachowania agenta.

RCKZOFRola-Cel-Kontekst-Zasady-Ograniczenia-Format

Struktura dobrej instrukcji agenta: kim jest, po co działa, co wie, jak działa, czego nie wolno i jak ma odpowiadać.

Cel agentaAgent goal

Jasno określony rezultat, do którego agent dąży. Punkt wyjścia całego projektowania — bez celu nie ma agenta.

OgraniczeniaConstraints / guardrails

Granice, których agent nie może przekroczyć (np. nie podaje danych osobowych, nie obiecuje rabatów). Chronią przed błędami i nadużyciami.

Human-in-the-loopCzłowiek w pętli

Zasada, że człowiek nadzoruje agenta i zatwierdza kluczowe decyzje przed ich realizacją.

Narzędzia i no-code
LLMLarge Language Model

Duży model językowy (np. GPT, Claude, Gemini) — silnik, na którym działają agenci. Generuje tekst, analizuje, planuje.

Custom GPTWłasny GPT (ChatGPT)

Spersonalizowany agent w ChatGPT z własną instrukcją i wiedzą (plikami). Tworzysz bez kodu w kilka minut.

Projekt (Claude)Claude Project

Przestrzeń w Claude z własnymi instrukcjami i dołączoną wiedzą — odpowiednik Custom GPT.

Zapierno-code automation

Łączy aplikacje i automatyzuje przepływy pracy bez programowania: trigger → akcje.

Make.comno-code automation

Wizualne budowanie scenariuszy automatyzacji (dawniej Integromat) — moduły łączone w przepływ, z krokami AI.

n8nworkflow automation

Otwarte narzędzie do automatyzacji workflow, które można hostować u siebie. Elastyczne integracje i kroki AI.

TriggerWyzwalacz

Zdarzenie uruchamiające automatyzację (np. „nowy formularz”, „nowy mail”). Początek każdego scenariusza no-code.

WorkflowPrzepływ pracy

Sekwencja połączonych kroków: trigger → akcja 1 → krok AI → akcja 2. Serce automatyzacji end-to-end.

APIApplication Programming Interface

Sposób, w jaki aplikacje „rozmawiają” ze sobą. Pozwala agentowi korzystać z zewnętrznych narzędzi i danych.

WebhookWeb callback

Automatyczne powiadomienie wysyłane z jednej aplikacji do drugiej w momencie zdarzenia — częsty trigger w Make/Zapier.

Wiedza i bezpieczeństwo
RAGRetrieval-Augmented Generation

Technika, w której agent pobiera wiedzę z Twoich dokumentów przed odpowiedzią — mniej halucynacji, więcej konkretów.

Baza wiedzyKnowledge base

Zbiór dokumentów (cennik, FAQ, procedury), z których korzysta agent. Im lepsza baza, tym trafniejsze odpowiedzi.

HalucynacjaHallucination

Pewnie brzmiąca, ale nieprawdziwa odpowiedź AI. Dlatego kluczowa jest weryfikacja i human-in-the-loop.

Dane osobowe / RODOGDPR

Nie wolno wklejać pełnych danych osobowych klientów do publicznych narzędzi AI. Zawsze weryfikuj i minimalizuj dane.

TokenToken

Jednostka tekstu, którą przetwarza model. Wpływa na koszt i limit długości rozmowy z agentem.

MCPModel Context Protocol

Standard łączenia agentów z zewnętrznymi narzędziami i danymi w bezpieczny, uporządkowany sposób.

Prompt
Prompt — mapa zadań do automatyzacji
Jestem [TWOJA ROLA] w branży [BRANŻA].
Wypisz 10 powtarzalnych, czasochłonnych zadań w mojej pracy, które mógłby przejąć Agent AI.

Myślenie agentowe i projektowanie zadań

Cel: Napiszesz kompletną instrukcję (prompt systemowy) swojego pierwszego agenta według struktury Rola–Cel–Kontekst–Zasady–Ograniczenia–Format.

Kroki
Dobra instrukcja = dobry agent. 80% jakości agenta powstaje tutaj, zanim cokolwiek zbudujesz.
Prompt
Szablon — instrukcja systemowa agenta (RCKZOF)
ROLA: Jesteś [np. asystentem obsługi klienta] dla [firma / branża].
CEL: Twoim zadaniem jest [konkretny, mierzalny cel].
KONTEKST I WIEDZA: [produkty/usługi, ceny, ton marki, najczęstsze pytania].
ZASADY DZIAŁANIA: [krok po kroku jak działasz; jak zadajesz pytania doprecyzowujące].
OGRANICZENIA: Nie podajesz [danych poufnych / obietnic]. Gdy nie znasz odpowiedzi — [eskalujesz do człowieka].
FORMAT ODPOWIEDZI: [np. krótko, punktami, po polsku, ton przyjazny].

Budowa Agentów AI krok po kroku

Cel: Zbudujesz działającego agenta (Custom GPT w ChatGPT lub Projekt w Claude), dodasz mu wiedzę i przetestujesz na realnych pytaniach.

Kroki
Kolejność: cel i zadania → instrukcja i wiedza → testowanie → optymalizacja → wdrożenie.
Prompt
Prompt — scenariusze testowe agenta
Wygeneruj 5 trudnych pytań testowych, które mogą zadać moi klienci, aby sprawdzić, czy mój agent:
1) odpowiada poprawnie i zgodnie z wiedzą,
2) trzyma się zasad i ograniczeń,
3) wie, kiedy przekazać sprawę człowiekowi.

Dla każdego pytania dopisz, jakiej dobrej odpowiedzi oczekuję.

Agenci AI w pracy indywidualnej

Cel: Uruchomisz agentów do 3 codziennych zadań: research, analiza/raport oraz szkice komunikacji.

Kroki
Prompt
Prompt — research z weryfikacją
Jestem [ROLA]. Zbierz i streść najważniejsze informacje na temat: [TEMAT].
Podaj: 5 kluczowych faktów, 3 wnioski dla mojej pracy i 3 pytania, które warto jeszcze zbadać.
Zaznacz, które informacje wymagają weryfikacji u źródła.
Prompt
Prompt — analiza danych i raport
Oto dane: [WKLEJ DANE / OPIS].
Wykonaj analizę: 1) 3 najważniejsze wnioski, 2) co wymaga uwagi/ryzyka, 3) 3 konkretne rekomendacje działań. Odpowiedz zwięźle, punktami.
Prompt
Prompt — szkic maila
Napisz profesjonalny, uprzejmy e-mail po polsku.
Cel maila: [np. odpowiedź na zapytanie / follow-up].
Odbiorca: [kto]. Kontekst: [sytuacja].
Ton: ludzki, konkretny, bez korporacyjnych zwrotów. Dodaj jasne CTA.
